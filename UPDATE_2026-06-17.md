# Update — 2026-06-17 (v5.2.0)

Bug-fix + safety rebuild. Highlights: a critical TradeStation hotkey
order-misfire fix, Finnhub 403 handling, preset date re-anchoring, and a
dev-only data-dir override. Version bumped `5.1.0 → 5.2.0`; exe rebuilt.

## Hotkey order-misfire fix (critical)

- **Root cause:** `pyautogui.typewrite` produces capital letters by *holding
  Shift* around each letter. The scanner sent tickers UPPERCASE, so every
  capital became `Shift`+letter — which collides with **TradeStation Trade
  Bar** order-entry hotkeys, firing one order per capital letter and
  splitting the ticker across the order bar and the command line. (TitanX has
  no such hotkeys, so it was unaffected.)
- **Fix:** type the ticker **lowercase** in `hotkey.send_ticker` and the bulk
  `tradestation.py` bridge. No Shift on the wire → no modifier+key hotkey can
  fire; symbol entry is case-insensitive so search/load still works.
- **Diagnostics:** added `HOTKEY_FOCUS_DIAGNOSTICS` (default **off**) — logs
  the foreground + focused window (`GetGUIThreadInfo`) at each send phase.
- **Tooling:** new `hotkey_probe.py` (repo root) — standalone, staged
  (`click-only` / `type` / `full`) bisection probe; **SIM-mode only**.

## Finnhub fill

- A **403** (`FAIL_FORBIDDEN` — symbol not in the account's plan) is now a
  permanent per-ticker skip: routed to the Finnhub skip list (persisted at run
  end) and **excluded from the consecutive-block streak**, so out-of-plan
  symbols no longer trigger a backoff pause or rewind-and-retry.
- Generalized via `fill_framework.FillSpec.skip_failure_kinds`.

## Presets

- **Non-sequenced** presets now re-anchor on load: **End = most recent
  trading day**, **Start = End − saved window span** — a stale saved date can
  no longer scan a past day. (Timeframe checkboxes already derive their own
  start from End; this also keeps Custom-Range / single-window presets fresh.)
- **Sequenced-run** presets keep their exact saved start/end window.

## Dev / infrastructure

- `TRADE_SCANNER_FH_DATA_DIR` env var optionally points a source/dev run at an
  existing `scanner_data/` tree (e.g. `dist/scanner_data`) without rebuilding
  the exe. Cascades to every derived path. Package-specific name to avoid
  colliding with unrelated environment variables (which would otherwise
  redirect even the frozen exe's data dir).
